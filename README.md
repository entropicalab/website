# entrópica · website

source for [entropica-lab.com](https://entropica-lab.com).

energy-led tropical architecture studio. bilingual (en / es). built with [astro](https://astro.build) — static html, markdown for blog and case studies, no cms. deploys to github pages on every push to `main`.

---

## quick start

```sh
npm install
npm run dev      # local dev at http://localhost:4321
npm run build    # output to ./dist
npm run preview  # serve the build locally
```

node 22+ is required (matches astro's engine requirement).

## adding a new blog post

1. create a markdown file in `src/content/blog/en/<slug>.md` (and optionally `src/content/blog/es/<slug>.md` for spanish).
2. frontmatter shape:

   ```yaml
   ---
   title: 'your post title.'
   slug: your-post-slug
   locale: en
   eyebrow: '// the boring part'
   summary: a one-sentence summary used on the index page.
   date: 2026-06-15
   author: josé barría
   draft: false
   ---
   ```

3. body is plain markdown. headings, lists, blockquotes, and `code` all styled. push to `main` and it's live within a couple of minutes.

set `draft: true` to keep a post out of the build.

## adding a new project / case study

1. create a markdown file in `src/content/projects/en/<slug>.md` (and the mirror in `src/content/projects/es/<slug>.md`).
2. follow the shape of an existing project (e.g. `lotus-hall.md`) — the frontmatter has fields for hero metric, brief, approach, strategy stack, gallery, and closing quote.
3. for `featured: true`, the project shows on the homepage. `order` controls sort.
4. when you have real photos, drop them in `public/projects/<slug>/` and reference them in the gallery `src` fields.

## the contact form (formspree)

the form posts to `https://formspree.io/f/REPLACE_ME` in two places:

- `src/pages/contact.astro` (english)
- `src/pages/es/contact.astro` (spanish)

to wire it up for real:

1. sign up at [formspree.io](https://formspree.io) with `info@entropica-lab.com` as the destination.
2. create a form and copy its endpoint id (looks like `xqkrweol`).
3. replace `REPLACE_ME` with the id in both files.
4. push.

the free plan gives 50 submissions/month, which is plenty to start. honeypot + standard formspree spam filtering are already wired.

## github pages deploy

`.github/workflows/deploy.yml` runs on every push to `main`.

**one-time setup, on github:**

1. repo settings → pages → source: **github actions**.
2. that's it on the github side.

**custom domain (`entropica-lab.com`):**

the `public/CNAME` file already contains `entropica-lab.com`. github will detect it on the first successful deploy.

at the domain registrar, point the apex `entropica-lab.com` and the `www` subdomain at github pages:

| type   | name          | value                                                   |
| ------ | ------------- | ------------------------------------------------------- |
| `A`    | `@` (apex)    | `185.199.108.153`                                       |
| `A`    | `@` (apex)    | `185.199.109.153`                                       |
| `A`    | `@` (apex)    | `185.199.110.153`                                       |
| `A`    | `@` (apex)    | `185.199.111.153`                                       |
| `CNAME`| `www`         | `entropicalab.github.io`                                |

(github's current ip set — check [the docs](https://docs.github.com/en/pages/configuring-a-custom-domain-for-your-github-pages-site/managing-a-custom-domain-for-your-github-pages-site#configuring-an-apex-domain) before launch in case they've changed.)

dns can take 24h to propagate. once it resolves, enable **enforce https** in the pages settings.

## brand system

the site imports `src/styles/entropica.css` — the canonical brand stylesheet from the [brandbook handoff](../entropica-brandbook/). do not edit it for one-off page tweaks; per-page styles live in each page's `<style>` block.

room-mode rules to keep in mind:

- **roboto mono, weight 300**, for all body and headlines
- **400** only for eyebrows, units, uppercase, meta keys
- **never 500/600** in room mode
- **lowercase everything**, including h1
- **eyebrows** get a `//` prefix and uppercase letter-spacing
- on paper, **one amber accent per page** — usually the proof metric
- numbers always tabular: `font-feature-settings: 'tnum' 1, 'lnum' 1`
- never use the text wordmark pattern (`entr<span class="amp">ó</span>pica`); always use the logo image at `/entropica-dark.png`

## local dev cert note (windows only)

if `npm install` or `npm create astro` fails with `UNABLE_TO_VERIFY_LEAF_SIGNATURE`, it's a corporate / antivirus tls inspection issue. the fix:

```powershell
# generate a pem from windows' own cert store
$out = "$pwd\..\.windows-ca.pem"
$sb = New-Object System.Text.StringBuilder
Get-ChildItem Cert:\LocalMachine\Root, Cert:\CurrentUser\Root | Sort-Object Thumbprint -Unique | ForEach-Object {
  [void]$sb.AppendLine('-----BEGIN CERTIFICATE-----')
  [void]$sb.AppendLine([Convert]::ToBase64String($_.RawData, 'InsertLineBreaks'))
  [void]$sb.AppendLine('-----END CERTIFICATE-----')
}
Set-Content -Path $out -Value $sb.ToString() -Encoding ASCII

# tell npm to use it
npm config set cafile "$out"
```

then run `export NODE_EXTRA_CA_CERTS="$out"` (or `$env:NODE_EXTRA_CA_CERTS="..."` in powershell) before any npm/astro command. github actions runs on linux and doesn't need any of this — it's a local-dev issue only.
